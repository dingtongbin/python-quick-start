## 15.1 robots.txt 协议

### 什么是 robots.txt？

robots.txt 是网站根目录下的文本文件，用于告知爬虫哪些页面可以访问。

### 查看 robots.txt

```
https://www.example.com/robots.txt
```

### 示例

```txt
# 所有爬虫
User-agent: *
Disallow: /admin/
Disallow: /private/
Allow: /public/

# Google 爬虫
User-agent: Googlebot
Disallow: /no-google/

# 禁止所有
User-agent: BadBot
Disallow: /
```

### 解析 robots.txt

```python
from urllib.robotparser import RobotFileParser

def check_robots(url):
    """检查 URL 是否允许爬取"""
    rp = RobotFileParser()
    
    # 获取 robots.txt
    from urllib.parse import urlparse
    parsed = urlparse(url)
    robots_url = f"{parsed.scheme}://{parsed.netloc}/robots.txt"
    
    rp.set_url(robots_url)
    rp.read()
    
    # 检查是否允许
    return rp.can_fetch('*', url)

# 使用
if check_robots('https://example.com/page'):
    print("允许爬取")
else:
    print("禁止爬取")
```

### Scrapy 配置


## 15.2 版权法基础

### 受保护的内容

✅ **受版权保护：**
- 原创文章
- 图片、视频
- 数据库（有独创性）
- 软件代码

❌ **不受版权保护：**
- 事实数据
- 公共领域作品
- 简短标题
- 想法和方法

### 合理使用原则

在以下情况下可能构成合理使用：

1. **目的** - 教育、研究、评论
2. **性质** - 事实性 vs 创造性
3. **数量** - 少量 vs 大量
4. **影响** - 是否影响原作品市场价值

⚠️ **注意：** 合理使用是法律抗辩理由，不是绝对权利。

## 15.3 个人隐私保护

### 个人信息定义

**个人信息**包括：
- 姓名、身份证号
- 电话号码、邮箱
- 位置信息
- 生物识别信息
- 网络标识符

### 中国《个人信息保护法》

#### 核心原则

1. **合法正当必要** - 必须有合法目的
2. **最小化** - 只收集必要的信息
3. **知情同意** - 必须获得用户同意
4. **安全保障** - 采取安全措施保护数据

#### 爬虫注意事项

❌ **禁止行为：**
- 爬取个人敏感信息
- 未经同意收集个人信息
- 非法买卖个人信息
- 泄露个人信息

✅ **合规做法：**
- 仅爬取公开的非个人信息
- 匿名化处理数据
- 不存储个人身份信息
- 明确数据使用目的

## 15.4 GDPR 合规

### 什么是 GDPR？

《通用数据保护条例》（GDPR）是欧盟的数据保护法规，适用于处理欧盟公民数据。

### 关键要求

1. **数据主体权利**
   - 访问权
   - 删除权（被遗忘权）
   - 可携带权

2. **数据处理原则**
   - 目的限制
   - 数据最小化
   - 准确性
   - 存储限制

3. **违规处罚**
   - 最高 2000 万欧元或全球营业额 4%

### 对爬虫的影响

如果你爬取的数据可能包含欧盟公民信息：

✅ **必须：**
- 进行数据保护影响评估
- 实施隐私保护措施
- 提供数据删除机制
- 记录数据处理活动

## 15.5 服务条款（ToS）

### 法律效力

违反网站服务条款可能构成：
- 违约
- 未经授权访问（计算机欺诈）
- 侵犯知识产权

### 常见限制条款

```
典型的服务条款可能禁止：

- 自动化访问
- 数据挖掘
- 反向工程
- 商业用途
- 大规模爬取
```

### 建议

1. **阅读 ToS** - 爬取前仔细阅读
2. **联系网站** - 寻求官方 API 或许可
3. **尊重意愿** - 如果明确禁止，不要爬取
4. **咨询律师** - 商业项目务必咨询

## 15.6 计算机犯罪相关法律

### 中国法律

#### 《网络安全法》

- 禁止非法侵入网络
- 禁止非法获取数据
- 禁止干扰网络功能

#### 《数据安全法》

- 重要数据保护
- 数据出境安全
- 数据处理活动规范

#### 《刑法》相关条款

- **第 285 条** - 非法侵入计算机信息系统罪
- **第 286 条** - 破坏计算机信息系统罪
- **第 253 条** - 侵犯公民个人信息罪

### 美国法律

#### CFAA（计算机欺诈和滥用法）

- 未经授权访问计算机系统
- hiQ Labs v. LinkedIn 案例：公开数据可能不构成"未经授权"

### 判例参考

**hiQ Labs v. LinkedIn (2019)**
- 法院裁定：爬取公开数据不违反 CFAA
- 但：仍可能违反其他法律（版权、ToS 等）

## 15.7 数据使用规范

### 合法使用场景

✅ **通常允许：**
- 学术研究
- 新闻报道
- 公共利益
- 个人学习

❌ **通常禁止：**
- 商业竞争
- 身份盗窃
- 垃圾邮件
- 价格操纵

### 数据使用最佳实践

1. **注明来源** - 引用数据来源
2. **非商业使用** - 避免商业利用
3. **不重新发布** - 不要完整复制内容
4. **尊重版权** - 注明版权归属
5. **限时使用** - 不长期存储数据

## 15.8 道德准则

### 爬虫开发者道德守则

#### 1. 尊重网站运营者

- 控制爬取频率
- 避开高峰时段
- 不干扰正常服务
- 考虑服务器成本

#### 2. 透明和诚实

- 明确身份和目的
- 提供联系方式
- 如实说明数据用途

#### 3. 负责任的数据处理

- 保护数据安全
- 不滥用数据
- 及时删除不必要数据
-  anonymize when possible

#### 4. 社会责任

- 考虑社会影响
- 不助长违法行为
- 促进知识共享
- 维护网络生态

## 15.10 实际案例分析

### 案例 1：LinkedIn vs. hiQ Labs

**背景：** hiQ 爬取 LinkedIn 公开的用户资料

**争议点：** 是否违反 CFAA

**结果：** 最高法院支持 hiQ，认为公开数据可以自由访问

**启示：** 
- 公开数据的爬取可能有法律依据
- 但仍需注意其他法律问题

### 案例 2：Facebook vs. Power Ventures

**背景：** Power Ventures 为用户聚合 Facebook 数据

**争议点：** 违反 ToS 和 CFAA

**结果：** Facebook 胜诉

**启示：**
- 违反 ToS 可能导致法律责任
- 即使用户授权也可能不够

### 案例 3：中国某爬虫公司案

**背景：** 公司爬取多家网站数据用于商业产品

**结果：** 负责人被判刑

**原因：**
- 突破技术防护措施
- 获取非公开数据
- 商业牟利

**启示：**
- 商业爬虫风险更高
- 不要突破反爬措施
- 避免获取非公开数据

## 15.12 常见问题

### Q1: 爬取公开数据违法吗？

**A:** 不一定。取决于：
- 是否有其他法律保护（版权、商业秘密）
- 是否违反服务条款
- 爬取方式和目的
- 当地法律规定

### Q2: 我可以爬取竞争对手的数据吗？

**A:** 高风险行为。建议：
- 仅爬取真正公开的数据
- 不用于不正当竞争
- 咨询法律专家
- 考虑其他方式获取

### Q3: 学术研究的爬虫合法吗？

**A:** 通常更安全，但也要：
- 遵守 robots.txt
- 最小化数据收集
- 匿名化处理
- 发表时谨慎处理数据

### Q4: 如何获得爬取许可？

**A:** 
1. 查找官方 API
2. 联系网站所有者
3. 说明目的和范围
4. 签订书面协议
5. 遵守约定条件
