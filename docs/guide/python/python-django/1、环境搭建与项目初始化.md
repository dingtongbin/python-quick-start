# 第1章：环境搭建与项目初始化

本章将带你从零开始，完成 Django 5.2 LTS 开发环境的搭建，并创建你的第一个 Django 项目。

---

## 1.1 Python 3.10 环境准备

### 为什么选择 Python 3.10？

✅ **稳定可靠** - Python 3.10 是成熟稳定的版本
✅ **完全兼容** - Django 5.2 LTS 完美支持 Python 3.10
✅ **广泛支持** - 大多数第三方包都支持 Python 3.10
✅ **长期维护** - Python 3.10 安全支持到 2026 年 10 月

### 检查 Python 版本

```bash
# 检查 Python 版本
python --version
# 或
python3 --version

# 期望输出
Python 3.10.x
```

如果版本不是 3.10，请安装 Python 3.10。

## 1.2 安装 Python 3.10

### Windows 系统

#### 方法一：官方安装包（推荐）

1. 访问 [Python 官网下载页面](https://www.python.org/downloads/)
2. 找到 Python 3.10.x 版本（如 3.10.11）
3. 下载 **Windows installer (64-bit)**
4. 运行安装程序
5. **重要步骤**：
   - ✅ 勾选 "Add Python 3.10 to PATH"
   - 选择 "Install Now" 或自定义安装路径
6. 等待安装完成

#### 验证安装

```powershell
# 打开新的 PowerShell 窗口
python --version
# 应该输出: Python 3.10.x

pip --version
# 应该输出: pip x.x.x from ... (python 3.10)
```

#### 方法二：使用 pyenv-win（多版本管理）

```powershell
# 安装 pyenv-win
pip install pyenv-win --target %USERPROFILE%\.pyenv

# 配置环境变量（添加到系统 PATH）
# %USERPROFILE%\.pyenv\pyenv-win\bin
# %USERPROFILE%\.pyenv\pyenv-win\shims

# 安装 Python 3.10
pyenv install 3.10.11

# 设置为全局版本
pyenv global 3.10.11

# 验证
python --version
```

### macOS 系统

#### 方法一：使用 Homebrew（推荐）

```bash
# 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Python 3.10
brew install python@3.10

# 验证
python3.10 --version

# 创建别名方便使用
echo 'alias python="python3.10"' >> ~/.zshrc
echo 'alias pip="pip3.10"' >> ~/.zshrc
source ~/.zshrc

# 现在可以直接使用
python --version
```

#### 方法二：官方安装包

1. 访问 https://www.python.org/downloads/mac-osx/
2. 下载 Python 3.10.x macOS 64-bit installer
3. 双击安装
4. 按照提示完成安装

### Linux 系统（Ubuntu/Debian）

```bash
# 更新包列表
sudo apt update

# 安装必要的依赖
sudo apt install -y software-properties-common

# 添加 deadsnakes PPA（提供多个 Python 版本）
sudo add-apt-repository ppa:deadsnakes/ppa

# 安装 Python 3.10
sudo apt install -y python3.10 python3.10-venv python3.10-dev

# 验证
python3.10 --version

# 设置默认 Python（可选）
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3.10 1
sudo update-alternatives --config python

# 验证
python --version
```

### CentOS/RHEL

```bash
# 安装 EPEL 仓库
sudo yum install -y epel-release

# 安装 IUS 仓库（提供 Python 3.10）
sudo yum install -y https://repo.ius.io/ius-release-el7.rpm

# 安装 Python 3.10
sudo yum install -y python310 python310-pip python310-devel

# 验证
python3.10 --version
```

## 1.3 虚拟环境配置

### 为什么需要虚拟环境？

✅ **隔离依赖** - 每个项目有独立的包环境
✅ **避免冲突** - 不同项目可以使用不同版本的包
✅ **便于部署** - 可以准确复制生产环境
✅ **保持清洁** - 不污染系统 Python 环境

### 创建虚拟环境

```bash
# 创建项目目录
mkdir django-tutorial
cd django-tutorial

# 使用 Python 3.10 创建虚拟环境
python -m venv venv

# 或使用完整路径（如果 python 不是 3.10）
python3.10 -m venv venv
```

### 激活虚拟环境

#### Windows (PowerShell)

```powershell
# 首次使用可能需要设置执行策略
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 成功后会看到前缀
(venv) PS D:\django-tutorial>
```

#### Windows (CMD)

```cmd
venv\Scripts\activate.bat

REM 成功后
(venv) D:\django-tutorial>
```

#### macOS/Linux

```bash
source venv/bin/activate

# 成功后
(venv) $ 
```

### 验证虚拟环境

```bash
# 检查 Python 版本（应该是 3.10）
python --version
# Python 3.10.x

# 检查 pip 位置（应该在 venv 中）
which python
# Windows: where python
# 应该指向 venv 目录

which pip
# 应该指向 venv 目录
```

### 退出虚拟环境

```bash
deactivate

# 前缀消失，回到系统环境
$
```

### 虚拟环境最佳实践

#### 1. 将 venv 加入 .gitignore

```bash
# 创建 .gitignore 文件
echo "venv/" > .gitignore
echo "__pycache__/" >> .gitignore
echo "*.pyc" >> .gitignore
echo ".DS_Store" >> .gitignore
echo "db.sqlite3" >> .gitignore
echo ".env" >> .gitignore
```

#### 2. 使用 requirements.txt 管理依赖

```bash
# 导出当前环境的包
pip freeze > requirements.txt

# 在新环境中安装
pip install -r requirements.txt
```

#### 3. virtualenvwrapper（可选高级工具）

```bash
# 安装
pip install virtualenvwrapper

# 配置 ~/.bashrc 或 ~/.zshrc
export WORKON_HOME=$HOME/.virtualenvs
export VIRTUALENVWRAPPER_PYTHON=$(which python3.10)
source $(which virtualenvwrapper.sh)

# 使用
mkvirtualenv myproject      # 创建并激活
workon myproject            # 激活已有环境
deactivate                  # 退出
rmvirtualenv myproject      # 删除
lsvirtualenv                # 列出所有环境
```

## 1.6 创建第一个 Django 项目

### 使用 django-admin 创建项目

```bash
# 确保在虚拟环境中
# 创建名为 myproject 的项目
django-admin startproject myproject

# 或使用 python -m
python -m django startproject myproject
```

### 项目结构详解

```
myproject/                  # 项目根目录
├── manage.py              # 项目管理脚本
└── myproject/             # 项目配置目录
    ├── __init__.py        # Python 包标识
    ├── asgi.py            # ASGI 配置（异步支持）
    ├── settings.py        # 项目配置文件 ⭐
    ├── urls.py            # 主 URL 路由配置 ⭐
    └── wsgi.py            # WSGI 配置（部署用）
```

### 文件说明

#### manage.py

```python
#!/usr/bin/env python
"""Django's command-line utility for administrative tasks."""
import os
import sys

def main():
    """Run administrative tasks."""
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'myproject.settings')
    try:
        from django.core.management import execute_from_command_line
    except ImportError as exc:
        raise ImportError(
            "Couldn't import Django. Are you sure it's installed and "
            "available on your PYTHONPATH environment variable? Did you "
            "forget to activate a virtual environment?"
        ) from exc
    execute_from_command_line(sys.argv)

if __name__ == '__main__':
    main()
```

**作用：**
- 运行开发服务器
- 数据库迁移
- 创建应用
- 收集静态文件
- 等等...

#### settings.py（核心配置文件）

关键配置项：

```python
# 安全密钥（不要泄露！）
SECRET_KEY = 'django-insecure-xxx'

# 调试模式（开发时 True，生产时 False）
DEBUG = True

# 允许的主机
ALLOWED_HOSTS = []

# 已安装的应用
INSTALLED_APPS = [
    'django.contrib.admin',      # 管理后台
    'django.contrib.auth',       # 认证系统
    'django.contrib.contenttypes', # 内容类型
    'django.contrib.sessions',   # 会话
    'django.contrib.messages',   # 消息框架
    'django.contrib.staticfiles', # 静态文件
]

# 中间件
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]

# URL 配置
ROOT_URLCONF = 'myproject.urls'

# 模板配置
TEMPLATES = [...]

# WSGI 应用
WSGI_APPLICATION = 'myproject.wsgi.application'

# 数据库配置（默认 SQLite）
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}

# 密码验证
AUTH_PASSWORD_VALIDATORS = [...]

# 国际化
LANGUAGE_CODE = 'en-us'
TIME_ZONE = 'UTC'
USE_I18N = True
USE_TZ = True

# 静态文件
STATIC_URL = 'static/'

# 默认主键字段类型
DEFAULT_AUTO_FIELD = 'django.db.models.BigAutoField'
```

#### urls.py（URL 路由）

```python
"""
```

## 1.9 创建第一个应用

### Django 项目 vs 应用

- **项目（Project）** - 整个网站，包含多个应用
- **应用（App）** - 功能模块，可复用

### 创建应用

```bash
# 在项目根目录执行
python manage.py startapp blog
```

### 应用结构

```
blog/                        # 应用目录
├── __init__.py             # Python 包标识
├── admin.py                # 管理后台配置
├── apps.py                 # 应用配置
├── models.py               # 数据模型 ⭐
├── tests.py                # 测试文件
└── views.py                # 视图函数 ⭐
```

### 注册应用

编辑 `myproject/settings.py`，在 `INSTALLED_APPS` 中添加：

```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    
    # 添加你的应用
    'blog',  # 或者 'blog.apps.BlogConfig'
]
```

### 创建第一个视图

编辑 `blog/views.py`：

```python
from django.http import HttpResponse

def hello(request):
    return HttpResponse("Hello, Django 5.2!")
```

### 配置 URL

#### 方法一：在主 URL 配置中添加

编辑 `myproject/urls.py`：

```python
from django.contrib import admin
from django.urls import path
from blog.views import hello

urlpatterns = [
    path('admin/', admin.site.urls),
    path('hello/', hello, name='hello'),
]
```

#### 方法二：使用应用级别的 URL（推荐）

创建 `blog/urls.py`：

```python
from django.urls import path
from . import views

app_name = 'blog'

urlpatterns = [
    path('hello/', views.hello, name='hello'),
]
```

修改 `myproject/urls.py`：

```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('blog/', include('blog.urls')),
]
```

### 测试

访问 http://127.0.0.1:8000/blog/hello/

你应该看到：**Hello, Django 5.2!**

## 1.10 settings.py 常用配置

### 时区和语言

```python
# settings.py

# 设置为中文
LANGUAGE_CODE = 'zh-hans'

# 设置时区为上海
TIME_ZONE = 'Asia/Shanghai'

# 启用时区支持
USE_TZ = True
```

### 允许的主机

```python
# 开发环境
ALLOWED_HOSTS = ['localhost', '127.0.0.1']

# 生产环境（示例）
# ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']
```

### 静态文件配置

```python
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# 静态文件 URL
STATIC_URL = '/static/'

# 静态文件目录
STATICFILES_DIRS = [
    BASE_DIR / 'static',
]

# 收集静态文件的目标目录（生产环境）
STATIC_ROOT = BASE_DIR / 'staticfiles'
```

创建静态文件目录：

```bash
mkdir static
mkdir static/css
mkdir static/js
mkdir static/images
```

### 媒体文件配置

```python
# 媒体文件 URL
MEDIA_URL = '/media/'

# 媒体文件存储目录
MEDIA_ROOT = BASE_DIR / 'media'
```

创建媒体目录：

```bash
mkdir media
```

### 调试工具栏（开发环境）

```python
# settings.py

# 仅在 DEBUG=True 时启用
if DEBUG:
    INSTALLED_APPS.append('debug_toolbar')
    MIDDLEWARE.insert(0, 'debug_toolbar.middleware.DebugToolbarMiddleware')
    
    INTERNAL_IPS = [
        '127.0.0.1',
    ]
```

配置 URL（`myproject/urls.py`）：

```python
from django.conf import settings
from django.conf.urls.static import static

urlpatterns = [
    # ... 你的 URL
]

# 开发环境提供媒体文件
if settings.DEBUG:
    import debug_toolbar
    urlpatterns = [
        path('__debug__/', include(debug_toolbar.urls)),
    ] + urlpatterns
    
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

## 1.11 常见问题

### 问题 1：python 命令找不到

**解决方案：**

```bash
# Windows - 检查 PATH
echo $env:PATH

# 重新安装 Python，确保勾选 "Add to PATH"

# 或使用完整路径
C:\Python310\python.exe --version
```

### 问题 2：虚拟环境激活失败

**Windows PowerShell 执行策略错误：**

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

**macOS/Linux 权限问题：**

```bash
chmod +x venv/bin/activate
source venv/bin/activate
```

### 问题 3：端口被占用

```bash
# 错误信息：Error: That port is already in use.

# 解决方案 1：使用其他端口
python manage.py runserver 8080

# 解决方案 2：关闭占用端口的进程
# Windows
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# macOS/Linux
lsof -ti:8000 | xargs kill -9
```

### 问题 4：迁移错误

```bash
# 删除数据库文件重新开始
rm db.sqlite3  # macOS/Linux
del db.sqlite3  # Windows

# 重新迁移
python manage.py migrate
```

### 问题 5：导入错误

```bash
# 确保在虚拟环境中
# 检查 Python 路径
which python  # macOS/Linux
where python  # Windows

# 重新安装包
pip install django
```

### 练习 1：环境检查

创建一个检查脚本 `check_env.py`，验证以下内容：
- Python 版本是否为 3.10.x
- Django 是否已安装且版本为 5.2.x
- 是否在虚拟环境中

```python
import sys
import django

print(f"Python 版本: {sys.version}")
print(f"Django 版本: {django.get_version()}")
print(f"虚拟环境: {'是' if hasattr(sys, 'real_prefix') or (hasattr(sys, 'base_prefix') and sys.base_prefix != sys.prefix) else '否'}")
```
