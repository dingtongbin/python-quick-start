# 第八章：模块与包

Python 通过**模块**（Module）和**包**（Package）组织代码，实现代码复用、命名空间管理和项目结构化。本章介绍如何使用内置模块、创建自定义模块及管理第三方库。

------

## 8.1 导入模块：`import` 与 `from ... import ...`

### 基本导入方式

| 语法                      | 说明                           | 使用示例                                 |
| ------------------------- | ------------------------------ | ---------------------------------------- |
| `import module`           | 导入整个模块                   | `import math; print(math.sqrt(16))`      |
| `import module as alias`  | 导入并重命名                   | `import numpy as np`                     |
| `from module import name` | 导入模块中的特定对象           | `from math import sqrt; print(sqrt(16))` |
| `from module import *`    | 导入所有公有对象（**不推荐**） | `from math import *`                     |

### 示例对比

```python
# 方式1：完整导入
import random
print(random.randint(1, 10))

# 方式2：选择性导入
from random import randint
print(randint(1, 10))

# 方式3：重命名（避免命名冲突）
import datetime as dt
now = dt.datetime.now()
```

> 最佳实践：
>
> - 优先使用 `import module`（清晰表明来源）
> - 仅在频繁使用某函数时用 `from ... import ...`
> - **禁止使用 `from module import \*`**（污染命名空间，降低可读性）

## 8.2 标准库常用模块简介

Python 自带丰富的标准库，无需安装即可使用。

### `math`：数学运算

```python
import math

print(math.sqrt(25))      # 5.0
print(math.pi)            # 3.141592653589793
print(math.ceil(3.2))     # 4（向上取整）
print(math.floor(3.8))    # 3（向下取整）
```

### `random`：随机数生成

```python
import random

print(random.random())        # 0.0 ~ 1.0 的浮点数
print(random.randint(1, 10))  # 1~10 的整数（含边界）
print(random.choice(['a', 'b', 'c']))  # 随机选择元素
random.shuffle([1, 2, 3])     # 原地打乱列表
```

### `datetime`：日期与时间

```python
from datetime import datetime, timedelta

now = datetime.now()
print(now.strftime("%Y-%m-%d %H:%M"))  # 格式化输出

# 时间计算
future = now + timedelta(days=7)
print(future)
```

### `os`：操作系统接口

```python
import os

print(os.getcwd())           # 获取当前工作目录
print(os.listdir('.'))       # 列出目录内容
os.mkdir('new_folder')       # 创建目录（若不存在）
print(os.path.join('a', 'b')) # 跨平台路径拼接 → a/b 或 a\b
```

> 提示：更多标准库模块见 [Python 官方文档](https://docs.python.org/3/library/)

## 8.3 创建自定义模块

任何 `.py` 文件都是一个模块。

### 步骤

1. 创建文件 `utils.py`
2. 在其中定义函数或变量
3. 在其他文件中导入使用

#### 示例：`utils.py`

```python
# utils.py
def read_file(filename):
    """读取文件并返回内容"""
    with open(filename, 'r', encoding='utf-8') as f:
        return f.read()

def write_file(filename, content):
    """写入内容到文件"""
    with open(filename, 'w', encoding='utf-8') as f:
        f.write(content)

PI = 3.14159
```

#### 使用模块：`main.py`

```python
# main.py
import utils

utils.write_file("test.txt", "Hello from module!")
content = utils.read_file("test.txt")
print(content)

print(f"PI = {utils.PI}")
```

> 注意：模块名应为合法标识符（小写字母+下划线），且**不能与标准库模块同名**（如不要命名为 `random.py`）。

## 8.4 包（package）结构与 `__init__.py`

**包**是包含多个模块的目录，用于组织大型项目。

### 基本结构

```
myproject/
├── __init__.py      # 标识此目录为包（Python 3.3+ 可省略，但建议保留）
├── core.py
├── utils/
│   ├── __init__.py
│   ├── file_ops.py
│   └── string_ops.py
└── main.py
```

### `__init__.py` 的作用

- 标识目录为 Python 包
- 可控制 `from package import *` 的行为
- 可在导入包时执行初始化代码

#### 示例：`utils/__init__.py`

```python
# utils/__init__.py
from .file_ops import read_file, write_file
from .string_ops import clean_text

# 定义公开接口
__all__ = ['read_file', 'write_file', 'clean_text']
```

### 导入包内模块

```python
# 绝对导入（推荐）
from utils.file_ops import read_file

# 相对导入（仅在包内使用）
from .file_ops import read_file  # 在 utils 内部模块中使用
```

> 规范：
>
> - 使用**绝对导入**提高可读性
> - 避免深度嵌套包（不超过 3 层）
> - `__init__.py` 保持简洁

## 8.5 安装第三方库（`pip` 基础）

Python 生态依赖 `pip` 管理第三方库。

### 基本命令

| 命令                              | 说明           |
| --------------------------------- | -------------- |
| `pip install package`             | 安装最新版     |
| `pip install package==1.2.3`      | 安装指定版本   |
| `pip list`                        | 列出已安装包   |
| `pip show package`                | 显示包信息     |
| `pip uninstall package`           | 卸载包         |
| `pip install -r requirements.txt` | 从文件批量安装 |

### 示例：安装 `requests`

```bash
# 安装
pip install requests

# 在代码中使用
import requests
response = requests.get("https://httpbin.org/json")
print(response.json())
```

### 虚拟环境（强烈推荐）

避免全局污染，每个项目使用独立环境：

```bash
# 创建虚拟环境（Python 3.3+ 内置 venv）
python -m venv myenv

# 激活（Windows）
myenv\Scripts\activate
# 激活（Linux/macOS）
source myenv/bin/activate

# 安装包（仅在此环境中）
pip install flask

# 退出虚拟环境
deactivate
```

> 提示：将依赖写入 `requirements.txt` 便于协作：
>
> ```txt
> # requirements.txt
> requests==2.31.0
> flask==2.3.2
> ```
