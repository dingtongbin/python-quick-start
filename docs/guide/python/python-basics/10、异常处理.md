# 第十章：异常处理

程序运行时可能遇到错误（如文件不存在、除零、类型不匹配等）。Python 通过**异常处理机制**优雅地应对这些情况，避免程序崩溃。

------

## 10.1 什么是异常？常见内置异常类型

### 异常（Exception）

- 程序执行期间发生的**错误事件**
- 若未处理，程序将终止并显示 traceback
- 所有异常都是 `BaseException` 的子类，常用异常继承自 `Exception`

### 常见内置异常

| 异常类型            | 触发场景                         | 示例                   |
| ------------------- | -------------------------------- | ---------------------- |
| `ValueError`        | 值类型正确但值无效               | `int("abc")`           |
| `TypeError`         | 操作或函数应用于不适当类型的对象 | `"a" + 1`              |
| `FileNotFoundError` | 尝试打开不存在的文件             | `open("missing.txt")`  |
| `ZeroDivisionError` | 除数为零                         | `5 / 0`                |
| `IndexError`        | 序列索引超出范围                 | `[1,2][5]`             |
| `KeyError`          | 字典中找不到指定键               | `{"a":1}["b"]`         |
| `AttributeError`    | 对象没有指定属性                 | `"hello".foo()`        |
| `NameError`         | 使用未定义的变量                 | `print(undefined_var)` |

> 提示：可通过 `dir(__builtins__)` 查看所有内置异常。

------

## 10.2 `try` / `except` 基本结构

使用 `try...except` 捕获并处理异常。

### 基本语法

```python
try:
    # 可能出错的代码
    risky_operation()
except:
    # 处理所有异常（不推荐）
    handle_error()
```

### 示例：处理除零错误

```python
try:
    x = int(input("请输入被除数: "))
    y = int(input("请输入除数: "))
    result = x / y
    print(f"结果: {result}")
except:
    print("发生错误！")
```

> 警告：**不要使用裸 `except:`**，它会捕获包括 `KeyboardInterrupt` 在内的所有异常，导致程序无法正常退出。

## 10.3 捕获特定异常与多异常处理

### 捕获特定异常

```python
try:
    value = int(user_input)
except ValueError:
    print("输入不是有效整数！")
```

### 多异常处理

#### 方式一：多个 `except` 块

```python
try:
    f = open("data.txt")
    data = f.read()
    number = int(data)
except FileNotFoundError:
    print("文件未找到")
except ValueError:
    print("文件内容不是有效数字")
```

#### 方式二：元组捕获同类异常

```python
try:
    # ...
except (FileNotFoundError, PermissionError):
    print("无法访问文件")
```

### 获取异常对象

使用 `as` 关键字获取异常实例，用于记录详细信息：

```python
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"计算错误: {e}")  # 输出: 计算错误: division by zero
```

> 最佳实践：
>
> - **只捕获预期的异常**
> - **按从具体到通用的顺序排列 `except` 块**

## 10.4 `else` 与 `finally` 子句

### `else` 子句

- **仅在 `try` 块未发生异常时执行**
- 常用于放置依赖于 `try` 成功的操作

```python
try:
    f = open("config.txt")
except FileNotFoundError:
    print("配置文件缺失")
else:
    # 文件成功打开后读取
    content = f.read()
    f.close()
    process_config(content)
```

### `finally` 子句

- **无论是否发生异常都会执行**
- 常用于清理资源（如关闭文件、释放锁）

```python
f = None
try:
    f = open("data.txt")
    data = f.read()
    process(data)
except Exception as e:
    log_error(e)
finally:
    if f and not f.closed:
        f.close()  # 确保文件关闭
```

> 注意：若使用 `with` 语句管理资源（如文件），通常不需要 `finally`。

## 10.5 主动抛出异常（`raise`）

使用 `raise` 手动触发异常，用于验证输入或表示逻辑错误。

### 基本用法

```python
def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b

# 调用
try:
    result = divide(10, 0)
except ValueError as e:
    print(e)  # 输出: 除数不能为零
```

### 重新抛出异常

在处理异常后仍希望上层处理：

```python
try:
    risky_operation()
except SpecificError as e:
    log_error(e)      # 记录日志
    raise             # 重新抛出原异常
```

### 自定义异常（简介）

通过继承 `Exception` 创建专用异常：

```python
class InvalidAgeError(Exception):
    """年龄无效的自定义异常"""
    pass

def set_age(age):
    if age < 0 or age > 150:
        raise InvalidAgeError(f"年龄 {age} 超出合理范围")

# 使用
try:
    set_age(-5)
except InvalidAgeError as e:
    print(e)
```

> 建议：为业务逻辑创建有意义的自定义异常，提升代码可读性。
