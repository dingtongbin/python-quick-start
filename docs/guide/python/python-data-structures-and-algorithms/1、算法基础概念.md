## 1.1 什么是算法？为什么重要？

### 算法的定义

**算法**是解决特定问题的一系列明确、有限的步骤。简单来说，算法就是解决问题的方法和步骤。

**算法的特征：**
1. **输入**：有零个或多个输入
2. **输出**：至少有一个输出
3. **确定性**：每一步骤都有明确的定义
4. **有限性**：在有限步骤内结束
5. **可行性**：每一步都可以有效执行

### 生活中的算法示例

```python
# 示例1：找出一组数中的最大值
def find_max(numbers):
    """找出列表中的最大值"""
    if not numbers:
        return None
    
    max_value = numbers[0]
    for num in numbers[1:]:
        if num > max_value:
            max_value = num
    
    return max_value

# 测试
nums = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5]
print(f"最大值: {find_max(nums)}")  # 输出: 9
```

```python
# 示例2：判断一个数是否为素数
def is_prime(n):
    """判断一个数是否为素数"""
    if n < 2:
        return False
    if n == 2:
        return True
    if n % 2 == 0:
        return False
    
    for i in range(3, int(n**0.5) + 1, 2):
        if n % i == 0:
            return False
    
    return True

# 测试
print(f"7是素数吗? {is_prime(7)}")      # True
print(f"10是素数吗? {is_prime(10)}")    # False
```

### 为什么学习算法很重要？

1. **提高效率**：好的算法可以大幅减少运行时间和资源消耗
2. **解决问题**：许多复杂问题需要特定的算法才能有效解决
3. **面试必备**：技术面试中算法题占很大比重
4. **编程思维**：培养逻辑思维和抽象能力
5. **优化意识**：学会评估和优化代码性能

### 算法 vs 数据结构

- **数据结构**：组织和存储数据的方式（如数组、链表、树）
- **算法**：操作数据的步骤和方法（如排序、搜索）

**关系**：数据结构是算法的基础，算法是数据结构的应用。选择合适的数据结构可以让算法更高效。

## 1.2 时间复杂度分析（Big O）

### 什么是时间复杂度？

时间复杂度描述算法运行时间随输入规模增长的变化趋势。我们关心的是**增长率**，而不是具体的运行时间。

### 为什么用 Big O 表示法？

Big O 表示法关注**最坏情况**下的时间复杂度，提供了一个上界保证。

**常见的时间复杂度（从快到慢）：**

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数时间 | 访问数组元素 |
| O(log n) | 对数时间 | 二分搜索 |
| O(n) | 线性时间 | 遍历数组 |
| O(n log n) | 线性对数时间 | 快速排序、归并排序 |
| O(n²) | 平方时间 | 冒泡排序、选择排序 |
| O(n³) | 立方时间 | 三层嵌套循环 |
| O(2ⁿ) | 指数时间 | 递归求解斐波那契数列 |
| O(n!) | 阶乘时间 | 全排列问题 |

### 复杂度分析示例

#### O(1) - 常数时间

```python
def get_first_element(arr):
    """获取数组第一个元素 - O(1)"""
    return arr[0] if arr else None

# 无论数组多大，都只需一步操作
```

#### O(n) - 线性时间

```python
def sum_array(arr):
    """计算数组元素之和 - O(n)"""
    total = 0
    for num in arr:  # 遍历所有元素
        total += num
    return total

# 数组长度增加一倍，运行时间也增加一倍
```

#### O(n²) - 平方时间

```python
def bubble_sort(arr):
    """冒泡排序 - O(n²)"""
    n = len(arr)
    for i in range(n):  # 外层循环 n 次
        for j in range(0, n-i-1):  # 内层循环 n-i-1 次
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

# 两层嵌套循环，总操作次数约为 n²/2
```

#### O(log n) - 对数时间

```python
def binary_search(arr, target):
    """二分搜索 - O(log n)"""
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1

# 每次排除一半的元素，所以是对数级别
```

### 如何计算时间复杂度？

**规则：**
1. **忽略常数**：O(2n) → O(n)
2. **保留最高阶**：O(n² + n) → O(n²)
3. **嵌套循环相乘**：两层循环 → O(n²)
4. **并列循环相加**：两个独立循环 → O(n + n) → O(n)

```python
def example_complexity(n):
    # O(1) - 常数操作
    a = 1
    
    # O(n) - 单层循环
    for i in range(n):
        print(i)
    
    # O(n²) - 双层嵌套循环
    for i in range(n):
        for j in range(n):
            print(i, j)
    
    # 总复杂度: O(1) + O(n) + O(n²) = O(n²)
```

## 1.3 空间复杂度分析

### 什么是空间复杂度？

空间复杂度描述算法所需内存空间随输入规模增长的变化趋势。包括：
- **输入数据**占用的空间
- **额外空间**（变量、数据结构等）

通常我们关注**额外空间**。

### 常见空间复杂度

#### O(1) - 常数额外空间

```python
def find_max(arr):
    """找出最大值 - O(1) 空间"""
    max_val = arr[0]  # 只使用一个额外变量
    for num in arr[1:]:
        if num > max_val:
            max_val = num
    return max_val
```

#### O(n) - 线性额外空间

```python
def duplicate_array(arr):
    """复制数组 - O(n) 空间"""
    new_arr = []  # 创建新数组
    for num in arr:
        new_arr.append(num)
    return new_arr
```

```python
def two_sum(nums, target):
    """两数之和 - O(n) 空间"""
    hash_map = {}  # 哈希表存储已遍历的元素
    for i, num in enumerate(nums):
        complement = target - num
        if complement in hash_map:
            return [hash_map[complement], i]
        hash_map[num] = i
    return []
```

#### O(n²) - 平方额外空间

```python
def create_matrix(n):
    """创建 n×n 矩阵 - O(n²) 空间"""
    matrix = [[0] * n for _ in range(n)]
    return matrix
```

### 时间与空间的权衡

很多时候需要在时间和空间之间做权衡：

```python
# 方法1：时间换空间 - O(n²) 时间，O(1) 空间
def contains_duplicate_slow(nums):
    """暴力法检查重复 - 慢但省空间"""
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] == nums[j]:
                return True
    return False

# 方法2：空间换时间 - O(n) 时间，O(n) 空间
def contains_duplicate_fast(nums):
    """哈希表法检查重复 - 快但费空间"""
    seen = set()
    for num in nums:
        if num in seen:
            return True
        seen.add(num)
    return False

# 测试
nums = [1, 2, 3, 4, 5, 3]
print(f"有重复吗? {contains_duplicate_fast(nums)}")  # True
```

## 1.5 算法设计策略简介

### 常见算法设计策略

#### 1. 分治法（Divide and Conquer）

将大问题分解为小问题，分别解决后合并结果。

**典型应用：**
- 归并排序
- 快速排序
- 二分搜索

```python
def merge_sort(arr):
    """归并排序 - 分治法"""
    if len(arr) <= 1:
        return arr
    
    # 分解
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    # 合并
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result

# 测试
arr = [38, 27, 43, 3, 9, 82, 10]
print(f"排序前: {arr}")
print(f"排序后: {merge_sort(arr)}")
```

#### 2. 动态规划（Dynamic Programming）

将问题分解为重叠子问题，保存子问题的解避免重复计算。

**典型应用：**
- 斐波那契数列
- 背包问题
- 最短路径

```python
def fibonacci_dp(n):
    """斐波那契数列 - 动态规划"""
    if n <= 1:
        return n
    
    dp = [0] * (n + 1)
    dp[1] = 1
    
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    
    return dp[n]

# 测试
print(f"F(10) = {fibonacci_dp(10)}")  # 55
```

#### 3. 贪心算法（Greedy）

每一步都做出局部最优选择，希望得到全局最优解。

**典型应用：**
- 活动选择问题
- 最小生成树
- Dijkstra 最短路径

```python
def activity_selection(activities):
    """活动选择问题 - 贪心算法"""
    # 按结束时间排序
    activities.sort(key=lambda x: x[1])
    
    selected = [activities[0]]
    last_end = activities[0][1]
    
    for start, end in activities[1:]:
        if start >= last_end:
            selected.append((start, end))
            last_end = end
    
    return selected

# 测试
activities = [(1, 3), (2, 5), (4, 7), (1, 8), (5, 9), (8, 10)]
print(f"可选择的活动: {activity_selection(activities)}")
```

#### 4. 回溯法（Backtracking）

尝试所有可能的解，发现不满足条件时回退。

**典型应用：**
- N皇后问题
- 数独求解
- 组合问题

```python
def permutations(nums):
    """全排列 - 回溯法"""
    result = []
    
    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return
        
        for i in range(len(remaining)):
            path.append(remaining[i])
            backtrack(path, remaining[:i] + remaining[i+1:])
            path.pop()
    
    backtrack([], nums)
    return result

# 测试
print(f"[1,2,3]的全排列: {permutations([1, 2, 3])}")
```

### 如何选择算法策略？

**决策流程：**

1. **问题是否可以分解为独立子问题？**
   - 是 → 分治法
   
2. **子问题是否重叠？**
   - 是 → 动态规划
   
3. **局部最优是否能导致全局最优？**
   - 是 → 贪心算法
   
4. **需要尝试所有可能解？**
   - 是 → 回溯法

### 练习 3：优化代码

优化以下代码的时间复杂度：

```python
# 原始代码 - O(n²)
def has_pair_sum_slow(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return True
    return False

# 优化后 - O(n)
def has_pair_sum_fast(nums, target):
    seen = set()
    for num in nums:
        complement = target - num
        if complement in seen:
            return True
        seen.add(num)
    return False
```
